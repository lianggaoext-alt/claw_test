## 未来迭代（已确认）

### 系统级硬隔离（非 root 运行 + sandbox/container）
目标：把当前“应用层隔离”升级为“系统级隔离”，降低越权风险。

范围：
1. 非 root 运行
   - bridge 服务改为低权限系统用户运行
   - 收紧 systemd 权限（NoNewPrivileges / ProtectSystem / ProtectHome 等）
2. sandbox 隔离
   - 限制可访问目录、系统调用和网络出口
3. container 隔离（生产推荐）
   - 每用户独立容器或独立运行实例
   - 仅挂载用户自己的 workspace

验收标准：
- 授权用户无法访问 `/root/.openclaw/workspace` 等非授权目录
- 未授权用户继续收到“你暂未开通权限，请联系管理员。”
- 现有企微消息收发链路保持可用

### 企微语音消息兼容（规划，未实现）
目标：支持企业微信语音消息进入 Agent 流程，并给出稳定回复。

范围（分阶段）：
1. 阶段1（流量验证）
   - 识别 `MsgType=voice`
   - 先返回“已收到语音，正在转写中（测试版）”
   - 打通语音分流与日志，不影响文本主链路
2. 阶段2（核心能力）
   - 根据 `MediaId` 下载语音媒体
   - 接入 ASR（语音转文字）
   - 转写文本进入现有 OpenClaw 流程
   - 主动推送最终回复（先文本）
3. 阶段3（体验增强）
   - 可选 TTS 语音回复
   - 支持用户偏好（文本优先/语音优先）

待决策项：
- ASR 供应商选型（腾讯云/阿里云/OpenAI Whisper/本地Whisper）
- 回复形式（仅文本 or 文本+语音）
- 时延目标（建议首条反馈 5~10 秒）

验收标准：
- 语音消息可稳定识别并进入 Agent
- 失败可回退到清晰提示（不静默失败）
- 不降低现有文本消息稳定性
